I have a list of songs. For each song, generate a JSON object containing descriptive tags and a unified "embedding text."

**Instructions for Analysis:**
Mentally estimate the following Spotify Audio Features for each track and use them to construct the output:
1. **Valence:** Is it happy/cheerful (High) or sad/depressed (Low)?
2. **Energy:** Is it intense/active (High) or calm/silent (Low)?
3. **Danceability:** Is it suitable for dancing (High) or irregular/slow (Low)?
4. **Tempo:** Is it fast-paced (High BPM) or slow (Low BPM)?
5. **Acousticness:** Is it acoustic/organic (High) or electronic/synthesized (Low)?
6. **Instrumentalness:** Is it instrumental (High) or vocal-heavy (Low)?
7. **Liveness:** Does it sound like a live performance?
8. **Lyrical Mood:** From the `lyrics_snippet` (if present), distill 2-3 short mood/theme labels describing how the words *feel* (e.g. "wistful", "defiant", "nostalgic"). If no lyrics are present, infer from the artist/title. Do NOT copy raw lyric phrases.

**Input Data:**
{songs_list}

Each song entry may include:
- `lastfm_tags`: Community-generated tags from Last.fm (use for mood, era, regional style)
- `lyrics_snippet`: A short excerpt of lyrics (use for mood, language, thematic content)

Use whichever signals are present. If a field is null or empty, rely on your musicological knowledge of the artist and title.

**How to write `embedding_text`:**
This string is embedded and compared against event-vibe queries, so it must maximize *vibe contrast* between songs — not read like a uniform template.
- **Lead with the most discriminating attributes:** start with energy level and mood/valence, then tempo, then danceability/acousticness, then genre. The first words carry the most weight.
- Be specific and concrete ("frantic, aggressive energy"; "sparse, melancholic and intimate") rather than generic ("a good song with nice energy").
- Do NOT open every entry with the same boilerplate stem (e.g. "A high-energy, fast-paced X song that..."). Vary the wording so two different-feeling songs produce clearly different text.
- Keep it to 1-2 sentences focused purely on how the song *feels and sounds*. Do NOT include the title, artist, or any lyric phrases.

**Required Output Schema (JSON):**
[
  {
    "id": 1,
    "title": "Song Name",
    "artist": "Artist Name",
    "vibe_tags": ["tag1", "tag2", "tag3", "tag4", "tag5"],

    "embedding_text": "Frantic, high-energy and aggressive, with a defiant, triumphant mood. Fast-paced and driving, built for hype rather than dancing. Hard-hitting electronic hip-hop with dense, forceful vocals."
  }
]

Return ONLY the JSON array above filled with real data for the input songs. No explanations, no code, no markdown prose — just the JSON array.
