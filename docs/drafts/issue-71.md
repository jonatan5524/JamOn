# 1. Introduction

## 1.2. Problem Statement

Music at social gatherings presents a fundamental coordination problem. When multiple individuals convene for an event—whether a study session, a party, or a road trip—selecting music that satisfies the collective taste of the group requires either imposing one person's preferences on everyone else or resorting to a lowest-common-denominator compromise. Research on group recommendation systems consistently identifies this tension as the preference aggregation problem: diverse individual tastes make it difficult to construct a shared listening experience that is meaningful to all participants [1]. In group contexts with even modest participant diversity, any single-user playlist risks alienating a significant portion of the audience.

Existing commercial tools fail to address this problem adequately. Spotify DJ operates exclusively within a single user's listening profile, applying AI-driven curation only to that individual's account, with no mechanism for incorporating the tastes of other attendees or the semantic nature of the occasion. Spotify Jam allows multiple users to contribute tracks to a shared queue, but functions purely as a passive aggregation point: it applies no vibe coherence, no contextual reasoning, and no event-awareness. Crucially, neither tool accepts a natural-language event description—such as "late-night study session" or "summer rooftop party"—as a meaningful signal to guide music selection. No existing platform combines a user's personal listening history with an open-ended, human-readable event description to generate a thematically coherent playlist. This gap between user intent and available tooling defines the core problem this project addresses.
